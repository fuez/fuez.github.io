CircleCI support s skipping builds

CircleCI supports the [ci skip] or [skip ci] standard for ignoring builds. When code changes are pushed to your VCS provider (GitHub or Bitbucket), if the last commit has [ci skip] or [skip ci] anywhere in the commit message title or description, then none of the commits in that push will be built automatically by CircleCI.
